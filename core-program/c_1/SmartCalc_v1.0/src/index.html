SmartCalc_v1.0

Особенности:

  • Музыкальное сопровождение;
  • Гифф анимация;
  • Наличие качественного оформления фронтенда;
  • Наличие валидации и проверок ошибок;
  • Наличие подсказок.
    Данный калькулятор позволяет вычислять любые инфиксные выражения содержащие в себе:
  • левую скобку - '(';
  • правую скобку - ')';
  • правую скобку - '+';
  • правую скобку - '-';
  • правую скобку - '*';
  • правую скобку - '/';
  • правую скобку - '^' (в алгоритме дейкстра не описано как работать с правоассоциативные операциями, также многие сайты обратной польской нотации используют возведение в степень как левоассоциативную операцию, в связи с этим данная операция является аналогом pow() *возможно, именно, поэтому в C нет возведения в степень* , то есть является левоассоциативной);
  • остаток от деления - 'mod' (в коде использовался fmod, так что работает со всеми типами чисел);
  • унарный плюс - '+' (если брать самый высокий приоритет унарных знаков, то алгоритм дейкстры даст различный результат в отличие от правильного, в связи с этим приоритет был выбран эмперическим путём, что способствовало правильной работе);
  • унраный минус - '-'.
    А также следующие функции:
  • вычисление косинуса - 'cos(x);
  • вычисление синуса - 'sin(x)';
  • вычисление тангеса - 'tan(x)';
  • вычисление арккосинуса - 'acos(x)';
  • вычисление арксинуса - 'asin(x)';
  • вычисление арктангеса - 'atan(x)';
  • вычисление квадратного корня - 'sqrt(x)';
  • вычисление натурального логарифма - 'ln(x)';
  • вычисление десятичного логарифма - 'log(x)'.

Как работать с калькулятором

  • К сожаления, цели связанные с установкой приложения не работают в связи с тем, что были использованы библиотеки QT6, которые не имеют грамотного решения в Makefile;
  • Запуск приложения осуществляется в QT 6;
  • Альтернативный вариант запуск exe файла в папке application.

История создания:

Изначально весь код был написан на C, весь функционал + дополнительные задания были выполнены без проблем. Что касается дополнительных заданий, то они были реализованы в простом варианте, а именно:

    Во втором дополнительном задании:
  • Внутренности платежей хранились в списке, без вывода на экран. В лучшем случае возможен был вывод в консоль;
  • Тип кредита выбирался на основе флажка.
    В третьем дополнительном задании:
  • Срок размещения изначально предполагал количество месяцев;
  • Переодичность выплат определялась количеством месяцев;
  • Капитализация определялась по флажку;
  • Каждый элемент списков содержал поле - номер месяца действия.

В дальнейшем при переносе на QT, часть функционала было перенесенно на C++ для удобства. В первую очередь, это касается всех дополнительных заданий. Также были созданы функции в C++ файлах для удобного взаимодействия с C функционалом.

Особенности калькулятора:

    Данный калькулятор создан с использованием:
  • Языка программирования - C;
  • Языка программирования - C++;
  • Фреймворка для C++ - QT;
  • Средства автоматической сборки - Makefile;
  • Средства автоматической сборки - qmakefile;
  • Утилит ОС - Linux;
  • Утилит ОС - MacOS.
    Файлы, которые потенциально можно добавить в цель для Make:
  • Папка - view/debug;
  • Папка - view/release;
  • Файл - view/Makefile;
  • Файл - view/Makefile.Debug;
  • Файл - view/Makefile.Release;
  • Файл - view/.qmake.stash;
    Слабые места, которые стоит подправить:
  • Makefile должен быть полностью рабочим;
  • Возможно, дополнить проверки на ошибки;
  • Все циклы в C файлах должны иметь 1 выход (стоит проверить, исправить);
  • Все функции в C файлах должны быть не более 50 строк (стоит проверить, исправить);
